## 🎨 Design Philosophy

### For Developers: How to Use These Designs

1. **Start with DESIGN_SYSTEM_COMPLETE.md**
   - This is your single source of truth
   - All colors, fonts, spacing defined
   - Component specifications ready to code

2. **Implement Components First**
   - Build reusable components following the spec
   - Use Tailwind CSS with our custom config
   - Consider using shadcn/ui as base

3. **Then Build Pages**
   - Follow the page-specific designs (LOGIN_PAGE_DESIGN.md, etc.)
   - Use the components you built
   - Maintain consistency with design system

4. **Stay Consistent**
   - Don't invent new colors/sizes
   - Use the 8px spacing grid
   - Follow the component specs exactly
   - When in doubt, ask Luna (me!)

### Design Principles

1. **Professional & Trustworthy** - Finance software must feel secure
2. **Speed & Efficiency** - Cashiers need to work fast
3. **Clarity Over Complexity** - Simple wins
4. **Touch-First** - Designed for tablets and phones
5. **Accessible to All** - Works for everyone

## 📐 Quick Reference

### Colors
```
Primary: #3B82F6 (Blue)
Success: #10B981 (Green)
Warning: #F59E0B (Orange)
Error: #EF4444 (Red)
Text: #111827 (Gray 900)
Background: #FFFFFF (White)
```

### Typography
```
Font: Inter (Primary), Fira Code (Monospace)
Base size: 16px
Scale: 12px, 14px, 16px, 18px, 20px, 24px, 30px, 36px
Weights: 400 (Regular), 600 (Semibold), 700 (Bold)
```

### Spacing
```
Based on 8px grid
xs: 4px, sm: 8px, md: 16px, lg: 24px, xl: 32px
Component padding: 16px (md)
Card padding: 24px (lg)
Section padding: 32-48px (xl-3xl)
```

### Components
```
Button height: 40px (md), 48px (lg)
Input height: 40px or 48px
Border radius: 6px (buttons/inputs), 8px (cards)
Touch targets: Minimum 44px, recommended 48px
```
