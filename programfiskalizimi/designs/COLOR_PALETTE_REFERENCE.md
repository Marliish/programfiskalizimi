## 🎨 Tailwind CSS Classes

### Quick Reference for Developers

```jsx
/* Background Colors */
bg-primary-500     // #3B82F6 (Main brand)
bg-success-500     // #10B981 (Success)
bg-warning-500     // #F59E0B (Warning)
bg-error-500       // #EF4444 (Error)
bg-gray-50         // #F9FAFB (Page background)
bg-white           // #FFFFFF (Card background)

/* Text Colors */
text-gray-900      // #111827 (Primary text)
text-gray-600      // #4B5563 (Secondary text)
text-gray-400      // #9CA3AF (Disabled text)
text-primary-500   // #3B82F6 (Links)

/* Border Colors */
border-gray-200    // #E5E7EB (Default border)
border-gray-300    // #D1D5DB (Input border)
border-primary-500 // #3B82F6 (Focus border)

/* Hover States */
hover:bg-primary-600     // Darker blue
hover:bg-success-600     // Darker green
hover:bg-error-600       // Darker red
hover:text-primary-600   // Darker blue text

/* Focus Rings */
focus:ring-primary-500   // Blue focus ring
focus:ring-error-500     // Red focus ring
focus:ring-success-500   // Green focus ring
```

## 🌈 Color Accessibility (WCAG)

### Text Contrast Ratios

✅ **Passes WCAG AA (4.5:1 minimum)**

**On White Background:**
- Gray 900 (`#111827`): 16.1:1 ✅ Excellent
- Gray 700 (`#374151`): 10.4:1 ✅ Excellent
- Gray 600 (`#4B5563`): 8.5:1 ✅ Excellent
- Primary 500 (`#3B82F6`): 4.9:1 ✅ Good (for large text)

**On Primary 500 Background:**
- White (`#FFFFFF`): 4.9:1 ✅ Good (for normal text)

**On Success 500 Background:**
- White (`#FFFFFF`): 3.3:1 ⚠️ Use for large text only

**On Error 500 Background:**
- White (`#FFFFFF`): 4.5:1 ✅ Good

Always test color combinations for accessibility!

## 💡 Usage Tips

### Do's ✅
- Use Primary 500 for main actions (buttons, links)
- Use Gray 900 for primary text
- Use Gray 600 for secondary text
- Use Success/Warning/Error for feedback messages
- Use light shades (50-100) for backgrounds
- Use dark shades (700-900) for text on light backgrounds

### Don'ts ❌
- Don't use random colors not in this palette
- Don't use Primary 400 or darker for backgrounds (too dark)
- Don't use Gray 400 or lighter for body text (low contrast)
- Don't mix different color families for same purpose
- Don't use too many colors on one screen (pick 2-3 max)

## 🎯 Color Decision Tree

**Need a color?** Follow this:

1. **Is it interactive?** → Primary 500 (Blue)
2. **Is it positive/success?** → Success 500 (Green)
3. **Is it a warning?** → Warning 500 (Orange)
4. **Is it an error/danger?** → Error 500 (Red)
5. **Is it text?** → Gray 900 (primary) or Gray 600 (secondary)
6. **Is it a background?** → White or Gray 50
7. **Is it a border?** → Gray 200 or Gray 300

**Still unsure?** Ask Luna (me)!

---

## 📋 Color Checklist

When implementing a component:
- [ ] Used only colors from this palette
- [ ] Text contrast passes WCAG AA (4.5:1)
- [ ] Hover state is darker shade of same color
- [ ] Focus state has visible ring (Primary 500)
- [ ] Error state uses Error 500
- [ ] Success state uses Success 500
- [ ] Disabled state uses Gray 300/400
